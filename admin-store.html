Configuration

🌐 URL du backend
L'URL de ton serveur Flask. En local : http://localhost:5000 · En production : URL de Render ou Railway.
💳 FedaPay
La clé publique est utilisée côté client (store.html). La clé secrète doit être dans les variables d'environnement de ton backend uniquement.
⚠️ La clé secrète ne doit JAMAIS être dans le code HTML. Elle va uniquement dans la variable d'environnement FEDAPAY_SECRET_KEY de ton backend.
📧 SMTP Email (Gmail)
Pour envoyer les emails de livraison. Gmail : active "Mot de passe d'application" dans les paramètres de sécurité.
Ces valeurs vont dans les variables d'environnement : SMTP_EMAIL et SMTP_PASSWORD. Configure-les dans Render/Railway.
🚀 Comment déployer le backend sans Fly.io
Option 1 — Render (gratuit)
1. Va sur render.com → New Web Service
2. Connecte ton GitHub (learnova-backend)
3. Build Command: pip install -r requirements.txt
4. Start Command: python app.py
5. Ajoute les variables d'env: FEDAPAY_SECRET_KEY, SMTP_EMAIL, SMTP_PASSWORD

Option 2 — Railway (encore plus simple)
1. railway.app → New Project → Deploy from GitHub
2. Il détecte Python automatiquement
3. Ajoute les variables d'env dans Settings → Variables

Option 3 — PythonAnywhere (complètement gratuit)
1. pythonanywhere.com → compte gratuit
2. Web → Add new web app → Flask
3. Uploads les fichiers, configure les variables

Option 4 — Tester en local (développement)
pip install flask flask-cors requests
FEDAPAY_SECRET_KEY=sk_sandbox_... python app.py
